### 学习内容

#### 1. 项目架构理解

**LipFD 的三大组件**：
1. **CLIP ViT 编码器**：提取全局特征
   - 原版使用 ViT-L/14（大模型，768维）
   - 作用：理解整张脸的语义信息
   
2. **Region Awareness 模块**：提取局部特征
   - 使用 ResNet50
   - 作用：关注嘴巴、头部等关键区域
   - 通过注意力机制加权融合
   
3. **分类器**：判断真假
   - 简单的全连接层
   - 输出：0（真）或 1（假）

**数据流**：
```
输入：视频（5帧）+ 音频
  ↓
CLIP ViT 编码器 → 全局特征 (768维)
  ↓
Region Awareness → 局部特征 + 注意力权重
  ↓
特征融合 → 分类器 → 真/假
```

**我的理解**：
- LipFD 的核心思想是检测音视频的时序不一致性
- 不仅看嘴巴，还看头部姿态（因为说话时头部也会动）
- 使用 CLIP 是因为它包含丰富的语义信息

**疑问**：
- Q: 为什么要用 CLIP 而不是普通的 CNN？
- A: CLIP 在大规模图文对上预训练，能理解语义信息，这对于理解唇形和音频的对应关系很重要

`models/LipFD.py` 的核心：
```python
class LipFD(nn.Module):
    def __init__(self, name, num_classes=1, device="cpu"):
        self.encoder, _ = clip.load(name, device=device)  # CLIP ViT
        self.backbone = get_backbone()                     # ResNet50
    
    def get_features(self, x):
        x = self.conv1(x)                    # 下采样
        features = self.encoder.encode_image(x)  # CLIP 提取特征
        return features
    
    def forward(self, x, feature):
        return self.backbone(x, feature)     # Region Awareness
```

**我的理解**：
- `get_features()` 用于提取全局特征（CLIP）
- `forward()` 用于提取局部特征并分类（Region Awareness）
- 两个步骤是分开的

### 遇到的问题与解决方案

#### 问题 1：不理解 Region Awareness 是如何工作的
**问题描述**：
- 代码中有很多 `weights`、`features_stack` 等变量，不知道是干什么的

**解决方案**：
- 阅读了 `region_awareness.py` 的代码
- 理解了它的工作流程：
  1. 对每个区域（嘴巴、头部等）提取特征
  2. 计算每个区域的注意力权重
  3. 用 softmax 归一化权重
  4. 加权融合所有区域的特征

**关键代码**：
```python
# 计算注意力权重
weights_stack = softmax(weights_stack, dim=2)

# 加权融合特征
parts.append(features_stack.mul(weights_stack).sum(2))
```

### 学习笔记

#### 关键概念

**1. 参数量（Parameters）**
- 定义：模型中所有可训练参数的数量
- 单位：M（百万）
- 意义：参数越多，模型越大，占用内存越多

**2. FLOPs（Floating Point Operations）**
- 定义：前向传播需要的浮点运算次数
- 单位：G（十亿）
- 意义：FLOPs 越大，计算量越大，推理越慢

**3. 推理时间（Latency）**
- 定义：处理一个样本需要的时间
- 单位：ms（毫秒）
- 意义：推理时间越短，模型越快

**4. 知识蒸馏（Knowledge Distillation）**
- 定义：用大模型（教师）指导小模型（学生）训练
- 方法：让学生学习教师的输出分布，而不仅仅是硬标签
- 优点：小模型可以达到接近大模型的性能

### 今天的思考

**Q: 为什么轻量化很重要？**
A: 因为大模型虽然准确率高，但在实际应用中（特别是移动端、实时场景）无法部署。轻量化可以在保持较高准确率的同时，大幅减少计算量和内存占用。

**Q: 轻量化的 trade-off 是什么？**
A: 参数量减少 → 模型容量减小 → 准确率可能下降。我们的目标是找到一个平衡点：用可接受的准确率损失，换取显著的速度提升。

**Q: 如何评价一个轻量化方案的好坏？**
A: 看三个指标：
1. 参数量减少了多少？（越多越好）
2. 推理速度提升了多少？（越快越好）
3. 准确率下降了多少？（越少越好）
